# 📡 Obsidian自动同步Skill
---
## 功能介绍
打通Claude Code开发会话和Obsidian知识库，自动同步代码库状态、开发日志、架构图到知识库的Johnny Decimal分类结构里，不用手动写项目文档。

---
## ⚙️ 配置方法
1. 先修改下面的知识库路径为你本地的实际路径：
```javascript
const VAULT_PATH = "/Users/yolanda/Documents/obsidian知识库/my-wiki-archive/"
```
⚠️ 重要：使用前一定要把上面的路径改成你自己Obsidian知识库的绝对路径！

2. 如果你的`~/.claude/commands/`目录还不存在，先创建：
```bash
mkdir -p ~/.claude/commands
```
3. 把这个文件复制到Claude命令目录：
```bash
cp obsidian-sync.md ~/.claude/commands/obsidian.md
```
4. 重启Claude Code就可以使用`/obsidian`命令了。

---
## 📖 使用说明
调用`/obsidian`命令后，会自动执行以下步骤：

### 阶段1：自动识别项目
首先检查`VAULT_PATH`路径是否存在，并且包含预期的目录结构（`00-09 系统配置`、`10-19 项目管理`等），如果路径不对会提示你修正。

然后自动识别当前项目：
- 把当前工作目录和已知项目路径匹配
- 检查`10-19 项目管理/11 进行中`目录下的项目文件，匹配`project/path`字段
- 检查`git remote -v`的GitHub地址
- 通过目录名或路径匹配知识库中的文件名

### 阶段2：分析项目信息
如果识别到已有项目：
- 读取项目的`package.json`/`pyproject.toml`等配置文件
- 扫描目录结构识别技术栈和模块
- 读取最近的Git提交记录
- 读取当前Claude会话的上下文信息

如果没有识别到项目，会提示你是否要创建新项目文档，自动分配下一个11.XX的项目ID。

### 阶段3：交互选择操作
识别到项目后会显示操作菜单供你选择：
| 操作选项 | 功能说明 |
| --- | --- |
| 🆕 创建新项目文档 | 自动生成完整的项目文档，包含技术栈、模块说明、架构图、开发日志 |
| 🔄 更新项目信息 | 重新扫描代码库，更新技术栈、模块、核心文件列表 |
| 📝 追加开发日志 | 把Git提交记录和会话上下文整理成开发日志追加到项目文档 |
| 🎨 生成架构画布 | 自动生成Obsidian Canvas格式的项目架构图 |
| 🔗 配置自动同步Hook | 给当前项目配置Git Hook，提交代码自动同步日志 |

### 阶段4：执行操作
执行你选择的操作，写入/更新Obsidian知识库文件，操作完成后会显示变更摘要，告诉你更新了哪些内容。

## 🔧 自定义配置
你可以根据自己的需求修改以下内容：
- 调整`PROJECT_TEMPLATE`变量：自定义生成的项目文档的结构和内容
- 调整`ARCHITECTURE_TEMPLATE`变量：自定义架构图的生成规则
- 调整`DEV_LOG_TEMPLATE`变量：自定义开发日志的格式
- 添加更多自定义操作选项：在菜单部分添加你需要的功能

---
## 🐛 常见问题
1. **提示路径不存在**：检查`VAULT_PATH`配置的路径是否正确，有没有拼写错误，必须是绝对路径。
2. **识别不到项目**：可以手动选择创建新项目，或者检查项目文档里的`project/path`字段是否和本地路径匹配。
3. **生成的内容不对**：可以手动修改Obsidian文档，下次更新的时候会保留你手动修改的内容，不会被覆盖。
4. **不想用了**：直接删除`~/.claude/commands/obsidian.md`文件即可，不会影响任何已有数据。
