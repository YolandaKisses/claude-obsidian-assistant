# DataDriver Fund QTrade Vue

基于 Vue 3 的量化交易前端平台，专注于股票分析、实时行情可视化与 AI 辅助决策。项目集成了 TradingView 专业图表、智能行情刷新策略、AI 股票分析模块和关注列表管理，为基金交易团队提供实时数据监控与交互式分析界面。目前处于活跃开发阶段，重点优化 AI 分析模块与行情数据性能。

## Architecture

**架构图：[[DataDriver Fund QTrade Vue Architecture.canvas]]**

### 核心系统

`用户操作` -> `前端路由` -> `状态管理` -> `API 服务层` -> `后端 Django REST API` -> `数据返回` -> `组件渲染` -> `可视化展示`

### 数据流
1. 用户通过 TradingView 图表或关注列表选择股票品种与周期
2. 前端组件触发状态更新，通过 Axios 调用后端 REST API
3. 后端处理行情数据请求、AI 分析计算与用户数据查询
4. 返回的 JSON 数据被状态管理库存储，驱动组件重新渲染
5. ECharts 与 TradingView 组件根据最新数据更新可视化图表
6. AI 分析模块使用 Markdown 渲染引擎展示结构化报告

### 模块划分
- **行情监控模块**：实时股票行情获取、智能刷新策略、交易时间判定
- **图表分析模块**：TradingView 专业图表集成、技术指标、周期切换
- **AI 分析模块**：股票 AI 分析报告生成、Markdown 渲染、历史记录管理
- **用户交互模块**：关注列表管理、自选股操作、手动刷新控制
- **系统配置模块**：环境变量管理、接口地址配置、多语言支持

- 实现 AI 分析频率与主图表周期的独立运行

### 功能修复与兼容性
- 修复 AI 分析模块复制功能在非 HTTPS 测试环境下失效的问题
- 增加 copyToClipboard 兼容性处理函数，引入 document.execCommand('copy') 作为 Clipboard API 的降级方案
- 解决 navigator.clipboard 在非安全上下文（如 HTTP 测试环境）下不可用的问题

### 行情刷新机制增强
- 在 rightContainer.vue 中实现智能动态随机刷新策略
- 废弃 `setInterval`，改用基于递归 `setTimeout` 的链式刷新
- 引入页面可见性感知：活跃状态 3-6min 随机刷新，后台状态 10-15min 随机刷新
- 实现即时唤醒机制：切回页面时自动触发补增刷新（由 `IS_WAKEUP_ENABLED` 常量控制）
- 完善生命周期管理：在 `onBeforeUnmount` 中销毁监听器与定时器，防止内存泄漏

### 交易时间逻辑优化
- 调整 isTradingTime 逻辑，支持 09:15-11:30 和 13:00-15:05 交易时段判定
- 提升行情轮询精度，非交易时段自动拦截接口调用，节省 API Token 及系统资源
