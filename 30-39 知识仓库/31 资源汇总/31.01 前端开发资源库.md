## 二、按场景查资料

### 1. 做页面开发

- 先确认项目是 Vue 2 还是 Vue 3
- 看对应组件库版本：`Element UI` 和 `Element Plus` 不可混用
- 查路由和状态管理时，确认是 `Vue Router 3/Vuex 3` 还是 `Vue Router 4/Pinia`

### 2. 做表格开发

- 普通管理后台表格：优先查 Element 表格能力
- 大表格、复杂表头、编辑表格：优先查 VXE Table
- 先看项目里已有封装组件，再决定是否直接看官方文档

### 3. 做图表和图形化

- 统计图：优先 ECharts
- 关系图/节点图：优先 X6
- 流程图：优先 BPMN-js
- 修改图表前，先确认项目里有没有通用封装组件可复用

### 4. 做联调

- 先看项目 `.env`、`vite.config.ts` 或 `config/index.js`
- 先确认代理、baseURL、鉴权头、超时配置
- 接口不通时，先区分是前端请求封装问题、跨域/代理问题，还是后端服务没起

## 三、当前项目的实用检查清单

### QTrade Vue 检查点

- 确认 `.env.development` 的 `VITE_API_URL`
- 确认页面入口在 `src/views/quantitative/*` 或 `src/views/AI/*`
- 确认图表逻辑是否复用了 `TVChartContainer` 或 ECharts 封装
- 高频刷新页面注意清理轮询和定时器

### DDAlg PingAn Vue 检查点

- 先确认真实入口文件，不要误改 `old`、`copy`、`backup`
- 先找页面目录，再回查 `src/api/*` 对应接口域
- 老项目页面和组件命名不统一时，优先搜中文业务词和接口文件名
- 流程图、X6、复杂表格问题通常都要同时看页面和公共组件

## 四、常用代码片段

### 1. 防抖函数

```javascript
function debounce(func, wait = 300) {
  let timeout
  return function() {
    clearTimeout(timeout)
    timeout = setTimeout(() => func.apply(this, arguments), wait)
  }
}
```

### 2. Excel 导出通用配置

```javascript
import XLSX from 'xlsx'
import FileSaver from 'file-saver'

export function exportExcel(data, fileName = '导出文件') {
  const ws = XLSX.utils.json_to_sheet(data)
  const wb = XLSX.utils.book_new()
  XLSX.utils.book_append_sheet(wb, ws, 'Sheet1')
  const wbout = XLSX.write(wb, { bookType: 'xlsx', type: 'array' })
  FileSaver.saveAs(new Blob([wbout]), `${fileName}.xlsx`)
}
```

### 3. Vue 2 图表销毁逻辑

export default {
  beforeDestroy() {
    if (this.chartInstance) {
      this.chartInstance.dispose()
      this.chartInstance = null
    }
  }
}

### 4. Vue 3 图表销毁逻辑

```typescript
import { onBeforeUnmount } from 'vue'

let chartInstance: any

onBeforeUnmount(() => {
  if (chartInstance) {
    chartInstance.dispose()
    chartInstance = null
  }
})
```

## 五、实践建议

- 优先看项目内已有实现，再看外部文档
- 任何库都先确认主版本，避免把 Vue 3 写法带进 Vue 2 项目
- 复杂页面修改前，先画出“页面 -> 组件 -> 接口”的最小链路
- 遇到旧项目大文件，先定位真实入口、主要接口、复用组件，再动手改
