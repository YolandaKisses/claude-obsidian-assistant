# Git Hook自动同步配置说明
实现Claude Code → Git → Obsidian 三者的自动数据流打通，提交代码自动同步开发上下文到知识库。

---
## ✅ 功能效果
每次执行`git commit`提交代码时，自动完成以下操作：
1. 自动把提交时间、提交哈希、提交信息写入对应Obsidian项目的开发日志
2. 自动追加本次提交对应的Claude会话链接，点击直接跳转到当时的开发上下文
3. 自动创建每日日志标题，不用手动整理格式

---
## 🔧 单项目配置（10秒搞定）
### 配置方法
1. 进入项目根目录
2. 复制Hook脚本到项目Git目录：
```bash
cp /你的知识库路径/claude-code/obsidian-sync/auto-sync-hook.sh .git/hooks/post-commit
```
3. 打开脚本修改开头2个配置项：
```bash
OBSIDIAN_PROJECT_FILE="对应Obsidian项目文档的绝对路径"
PROJECT_NAME="项目中文名"
```
4. 加执行权限：
```bash
chmod +x .git/hooks/post-commit
```
配置完成，提交代码会自动同步。

## 🌎 全局配置（一次配置所有新项目自动生效）
适合经常开新项目的场景，配置后所有新`git clone`/`git init`的项目自动带上同步功能：
1. 创建全局Git模板目录：
```bash
mkdir -p ~/.git-templates/hooks
```
2. 把`auto-sync-hook.sh`复制到模板目录：
```bash
cp /你的知识库路径/claude-code/obsidian-sync/auto-sync-hook.sh ~/.git-templates/hooks/post-commit
chmod +x ~/.git-templates/hooks/post-commit
```
3. 配置Git全局使用模板：
```bash
git config --global init.templatedir '~/.git-templates'
```
配置完成，以后所有新建/克隆的项目自动带同步功能，只需要修改每个项目里的`OBSIDIAN_PROJECT_FILE`和`PROJECT_NAME`配置即可。

## 🐛 常见问题排查
### 1. 提交代码后Obsidian日志没更新
- 检查`.git/hooks/post-commit`有没有执行权限：`ls -la .git/hooks/`看有没有x权限
- 检查`OBSIDIAN_PROJECT_FILE`路径是不是正确，文件是否存在
- 手动执行`.git/hooks/post-commit`看有没有报错信息

### 2. 会话ID是空的
- 确认你是在Claude Code内置的终端里提交的代码，外部终端无法获取`$CLAUDE_SESSION_ID`环境变量
- 升级到最新版Claude Code，老版本可能不支持这个环境变量

### 3. 不想用了怎么关闭
直接删除项目的`.git/hooks/post-commit`文件即可，完全不影响Git使用。
