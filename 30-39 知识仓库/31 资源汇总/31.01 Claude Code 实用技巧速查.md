# Claude Code 实用技巧速查

## 一、常用快捷键
| 操作 | 快捷键 | 说明 |
| --- | --- | --- |
| 快速选中代码 | `Cmd + L` | 逐行选中，连续按选中多行 |
| 多行编辑 | `Cmd + Option + 点击` | 同时编辑多个位置 |
| 清空当前会话 | `Cmd + K` | 快速清空会话上下文 |
| 开启计划模式 | `/plan` | 复杂需求开发前先做架构规划 |
| 停止当前任务 | `Cmd + C` | 中断正在运行的工具调用 |
| 搜索历史会话 | `Cmd + Shift + O` | 快速查找过往开发上下文 |

## 二、常用 Slash 命令
| 命令 | 用途 | 使用场景 |
| --- | --- | --- |
| `/help` | 查看帮助文档 | 忘记命令用法时 |
| `/commit` | 一键提交代码 | 开发完成后自动生成规范提交信息 |
| `/review` | 代码评审 | 提交PR前调用Agent自动评审代码问题 |
| `/obsidian` | 同步开发日志 | 手动触发会话记录同步到Obsidian项目 |
| `/fast` | 开启快速模式 | 简单需求/查文档时启用，响应更快 |
| `/exitplan` | 退出计划模式 | 架构方案确认后进入开发阶段 |

## 三、Agent 调用指南
按场景选择对应的Agent，提升效率：
| Agent类型 | 适用场景 |
| --- | --- |
| `feature-dev/code-reviewer` | 代码评审，检查bug、安全问题、性能优化点 |
| `feature-dev/code-architect` | 新功能开发前做架构设计、技术选型、模块拆分 |
| `feature-dev/code-explorer` | 刚接手陌生项目，快速理解代码结构、调用逻辑 |
| `claude-code-guide` | 遇到Claude Code使用问题、配置问题时调用 |
| `general-purpose` | 复杂多步骤任务、跨模块开发、需要多工具并行调用 |

## 四、常见问题处理
### 1. 会话丢失/上下文找不到
- 优先去`claude-code/sessions-backup`目录查找历史备份
- 按项目名/时间关键词搜索历史会话
### 2. Git Hook 不触发
- 执行`chmod +x .git/hooks/post-commit`给钩子加执行权限
- 手动执行`.git/hooks/post-commit`测试运行
### 3. Obsidian上下文不自动加载
- 检查项目根目录是否有`CLAUDE.md`文件
- 确认`/Users/yolanda/.claude/hooks/load-obsidian-context.sh`有执行权限
### 4. 权限提示要确认
- 基础读写/命令执行直接选允许即可
- 涉及删除/force push/远程服务器操作时先确认再允许
