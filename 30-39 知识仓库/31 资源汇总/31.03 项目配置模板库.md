## 二、.clauderules 规则模板（Vue 2 版）
```markdown
# 项目规则：[项目名]

## 1. 技术栈规范
- 遵循 Vue 2 Options API 写法，禁止使用 Vue 3 专属语法
- 优先使用 Element UI 现有组件，表格相关优先使用 VXE Table 封装组件
- 涉及可视化开发时，确保组件销毁时清除实例，避免内存泄漏
- Excel导入导出使用项目封装的excelUtil工具类

## 2. 业务规则
- 代码修改前优先阅读现有代码风格，保持统一
- 提交代码前必须走eslint校验，禁止提交有语法错误的代码
- 复杂功能开发前先做架构评审，确认方案后再开发
```

## 三、Git Hook 自动同步日志模板（post-commit）
```bash
#!/bin/bash
OBSIDIAN_FILE="[Obsidian项目文档路径]"
TODAY=$(date +"%B %d, %Y")
COMMIT_HASH=$(git rev-parse --short HEAD)
COMMIT_MSG=$(git log -1 --pretty=%B)
COMMIT_TIME=$(date +"%H:%M")

if [ ! -f "$OBSIDIAN_FILE" ]; then
    echo "⚠️  Obsidian 文件不存在，跳过更新"
    exit 0
fi

# 追加提交记录到开发日志
awk -v today="$TODAY" -v hash="$COMMIT_HASH" -v msg="$COMMIT_MSG" -v time="$COMMIT_TIME" '
/## 开发日志/ {
    print $0
    print ""
    print "### Dev Log — " today
    print "#### Git Commit (" time ") - " hash
    print "```"
    print msg
    print "```"
    next
}
1
' "$OBSIDIAN_FILE" > "$OBSIDIAN_FILE.tmp" && mv "$OBSIDIAN_FILE.tmp" "$OBSIDIAN_FILE"

echo "✅ Obsidian 开发日志已更新"
```
