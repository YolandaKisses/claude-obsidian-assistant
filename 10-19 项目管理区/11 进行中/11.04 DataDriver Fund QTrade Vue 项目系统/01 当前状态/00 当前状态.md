# 当前状态

## 摘要块
- 这篇讲什么：适合优先给人和 AI 看的项目现状快照。
- 当前结论：项目系统已初始化，当前重点是把真实业务模块、页面入口和 API 映射逐步补齐。
- 已过期内容：11.03 PingAn Vue 项目事实不作为本项目事实。
- 下一步看什么：开发任务优先读 [[00 模块地图总览]] 和 [[00 API 契约总览]]。

## 当前目标
- 建立当前仓库与 Obsidian 项目系统的一一关联。
- 让后续协作能快速定位量化交易页面、系统管理页面、通用组件和 API 调用入口。
- 将会话中反复出现的稳定结论升级为模块卡、页面卡、API 卡、问题卡或交付记录。

## 当前关注点
- 当前重点模块：`src/views/quantitative`、`src/views/system`、`src/components`、`src/api`、`src/router`、`src/stores`
- 当前聚焦页面 / 链路：量化交易页面、股票池、缠论、模拟基金、回测、实盘交易、AI 分析入口。
- 当前工程基线：Vue 3 + TypeScript + Vite + Element Plus + Pinia。
- 当前暂不处理：没有进入具体任务链路的历史 demo 页面和框架示例页面。

## 当前有效结论
- 本项目应以“业务页面目录 + 页面 API 文件 + 通用 API 域”组合定位。
- 量化交易相关页面集中在 `src/views/quantitative`。
- 系统后台基础能力集中在 `src/views/system`，并大量沿用 `django-vue3-admin` 框架约定。
- 组件入口集中在 `src/components`，其中图表、表格选择器、TradingView、股票 AI 分析等组件与量化交易需求相关。

## 当前风险
- README 与 `package.json` 仍保留大量上游 `django-vue3-admin` 信息，不能直接等同于当前业务事实。
- 页面与接口分散在页面目录内 `api.ts` 和公共 `src/api`，需要按页面持续补映射。
- 历史示例、demo、插件页面可能干扰真实业务入口判断。
- 当前骨架是初始版本，模块卡和 API 卡还需要随真实任务逐步加细。

## 当前待补
- 量化交易模块细分：股票池、缠论、模拟基金、回测、实盘交易、AI 分析。
- 页面到 API 的双向映射。
- 通用组件与业务页面引用关系。
- 真实问题排查卡和交付记录。

## 下一步建议
- 每次开发前先定位页面卡和 API 卡，缺卡时先补最小卡片。
- 每次有实际改动闭环后追加 [[10 运行上下文记录]]，并判断是否升级到结构化卡片。
- 对 README 中的上游框架内容保持“参考”态度，业务事实以当前仓库代码和本项目系统为准。
