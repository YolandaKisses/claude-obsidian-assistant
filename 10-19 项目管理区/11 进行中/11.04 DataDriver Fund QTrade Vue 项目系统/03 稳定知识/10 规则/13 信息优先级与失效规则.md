# 信息优先级与失效规则

## 默认优先级
1. [[00 当前状态]]
2. [[00 交付记录总览]] / [[00 问题排查总览]] 中最近仍有效的结论
3. [[00 稳定知识总览]] / [[00 决策记录总览]] / [[00 模块地图总览]] / [[00 API 契约总览]]
4. [[00 会话日志索引]] / [[10 运行上下文记录]]

## 冲突处理
- 如果当前状态与其他卡片冲突，以当前状态为准。
- 如果交付记录中有更近且已验证的结论，以交付记录为准。
- 如果日志内容未升级为稳定知识，默认视为过程参考。
- 未验证内容不得写成稳定事实。

## 失效规则
- 已不代表当前事实的内容应标记为历史参考或 `deprecated`。
- 旧结论被新交付覆盖时，保留历史原因，但不要继续作为当前事实引用。
