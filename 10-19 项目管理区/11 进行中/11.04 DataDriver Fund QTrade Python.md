# DataDriver Fund QTrade Python

量化交易平台的后端主服务，承担行情接口、量化选股、缠论分析、AI 报告生成、用户与权限管理等核心能力。该项目以 `backend/` 为主要开发区，当前是 `DataDriver Fund QTrade Vue` 的主要后端支撑。

## 项目信息

- **代码路径：** `/Users/yolanda/Documents/WorkSpace工作区/量化交易/datadriver-fund-qtrade-python`
- **技术栈：** Django 4.2 + DRF + Channels + Celery + MySQL + Redis
- **启动脚本：** `./start.sh`
- **服务地址：** `http://localhost:8000/`
- **主要开发目录：** `backend/`
- **前端对应项目：** [[11.02 DataDriver Fund QTrade Vue]]
- **架构图：** [[DataDriver Fund QTrade Python Architecture.canvas]]
- **上下文记录：** [[11.04 DataDriver Fund QTrade Python 上下文记录]]
- **最近整理：** 2026-04-20

## 当前定位

- 对外提供量化交易相关 REST API
- 承接缠论、选股、分析报告等核心业务逻辑
- 为前端页面提供 TradingView、行情、回测、股票池等数据接口
- 管理系统基础能力复用 dvadmin 体系

## Stack

| Layer | Tech | Purpose |
|-------|------|---------|
| Web 框架 | Django 4.2 + DRF 3.15 | API 服务与业务承载 |
| 实时通信 | Channels 4.1 | WebSocket / 实时能力 |
| 异步任务 | Celery | 后台任务与定时任务 |
| 数据库 | MySQL + Redis | 业务数据、缓存、消息队列 |
| 分析库 | Pandas / NumPy / TA-Lib / pandas-ta | 行情处理与指标计算 |
| AI 能力 | OpenAI API / LangChain | 分析报告生成与增强 |
| 量化能力 | czsc / tushare / backtesting | 缠论、数据获取与回测 |

## Architecture

**架构图：[[DataDriver Fund QTrade Python Architecture.canvas]]**

### 核心链路

1. **行情接口链路**：请求入站 -> 业务视图 -> 数据源/缓存 -> 数据清洗 -> 响应返回
2. **缠论分析链路**：K 线数据 -> 缠论计算 -> 信号/结构识别 -> AI 解释增强 -> 结果输出
3. **选股与回测链路**：条件配置 -> 任务执行 -> 结果存储 -> 前端查询/展示
4. **系统管理链路**：用户权限 -> 菜单/角色 -> 通用系统接口

### 当前仓库结构理解

项目说明里曾混合描述“理想结构”和“实际结构”，后续应以真实目录为准。目前主要关注：

- `backend/dvadmin/selection`：量化选股、交易分析相关业务
- `backend/core/chanlun`：缠论相关核心代码
- `backend/czsc`：缠论/量化分析库与扩展能力
- `backend/application`：Django 应用配置
- `backend/conf`：环境变量与项目配置

## 联调与排障关注点

- 先确认 `backend/venv` 是否存在，否则 `start.sh` 会直接退出
- 前端通常通过 `QTrade Vue` 仓库与本项目联调，而不是使用本仓库根目录下的 `web/` 目录
- 涉及行情、缠论、AI 分析时，优先定位 `selection`、`core/chanlun`、`czsc` 三块
- 涉及权限、菜单、登录时，优先定位 `dvadmin/system`
- 排查接口问题时，优先从 `urls.py -> views/*.py -> services/*.py` 这条线走
