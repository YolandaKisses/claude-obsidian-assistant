# DataDriver Fund DDAlg PingAn Vue

面向指标中心/平安场景的前端平台，承接指标配置、报表配置、产品配置、流程设计、数据上报与部分图形化关系展示能力。项目基于 Vue 2 与 Webpack 体系，模块量较大，页面命名存在一定历史包袱，因此文档需要更强调“模块入口”和“定位信息”。

## 项目信息

- **代码路径：** `/Users/yolanda/Documents/WorkSpace工作区/指标中心/datadriver-fund-ddalg-pingan-vue`
- **技术栈：** Vue 2 + Element UI + VXE Table + BPMN-js + AntV X6 + Webpack 4
- **开发命令：** `npm run dev`
- **开发端口：** `http://0.0.0.0:9527`（见 `config/index.js`）
- **生产构建：** `npm run build:prod`
- **SIT 构建：** `npm run build:sit`
- **架构图：** [[DataDriver Fund DDAlg PingAn Vue Architecture.canvas]]
- **上下文记录：** [[11.03 DataDriver Fund DDAlg PingAn Vue 上下文记录]]
- **最近整理：** 2026-04-20

## 当前项目特征

- 历史沉淀较深，目录和页面命名不完全统一
- 业务范围较宽，既有指标配置，也有报表、产品、流程和数据视图能力
- 图形化能力较强，涉及 BPMN 流程图与 X6/关系图组件
- 存在 `old`、`copy`、`backup` 等历史文件，修改前需要确认是否为真实入口
- 适合在文档中持续维护“页面名 -> 目录路径 -> 接口域”的映射关系

## 业务模块视角

### 核心模块

- **指标中心**：指标维护、指标分类、属性维护、配置相关页面
- **报表配置**：报表模板、报表配置、报表历史、数据上报
- **产品与组合**：产品配置、组合详情、侧边栏/弹窗交互
- **流程设计**：BPMN 流程图设计、流程审批配置
- **图谱与关系**：X6、关系图、可视化组件
- **数据集成**：ETL、XBRL、导入导出、数据修复类页面

### 典型交互链路

`页面视图` -> `src/api/* 业务域接口` -> `通用组件 / 表格 / 图形组件` -> `后端服务` -> `页面回显`

对这个项目来说，最重要的不是泛化架构描述，而是尽快定位“某个业务页面到底在哪个目录、依赖哪类组件、打哪组接口”。

## 开发时特别注意

- 看到 `old`、`copy`、`backup` 文件名时，不要默认它们是当前入口
- 这个项目里“一个页面目录下挂多个历史版本”的情况比较多，先从路由或菜单配置确认真实引用
- 搜页面时，优先从 `src/views` 和 `src/api` 成对查找，再看共享组件目录
- 查流程图/X6 类需求时，先从使用页面反查到组件，而不是直接在组件库里盲找
