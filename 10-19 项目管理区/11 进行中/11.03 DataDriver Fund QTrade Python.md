# DataDriver Fund QTrade Python

基于 Django 4.2 的量化交易后端服务，提供实时行情接口、TradingView 专业图表数据、AI 驱动的缠论技术分析与报告系统、策略回测框架和自动化交易能力，为基金交易团队提供完整的量化技术支撑。

**Location:** `/Users/yolanda/Documents/WorkSpace工作区/量化交易/datadriver-fund-qtrade-python`
**Port:** localhost:8000

---

## Stack

| Layer | Tech | Purpose |
|-------|------|---------|
| 框架 | Django 4.2 + Django REST Framework 3.15 | Web服务框架与RESTful API实现 |
| 语言 | Python 3.x | 后端开发语言 |
| 量化分析 | Pandas, NumPy, TA-Lib, pandas-ta | 行情数据处理与技术指标计算 |
| AI模块 | OpenAI API, LangChain | 缠论结构解析与智能分析报告生成 |
| 数据持久化 | MySQL, Redis, ClickHouse | 行情数据、用户数据、计算结果存储 |
| 实时通信 | Channels 4.1 | WebSocket实时行情推送 |
| 任务调度 | Celery | 异步任务与定时任务处理 |
| 数据接口 | Tushare, CCXT | 市场行情数据获取与交易所对接 |
| 可视化 | Plotly, Pyecharts, Matplotlib | 量化分析结果可视化 |
| 机器学习 | Scikit-learn, LightGBM, Optuna | 策略模型训练与参数优化 |

本项目采用前后端分离架构，前端使用Vue 3实现，后端提供完整的API服务与量化计算能力，支持高并发行情查询和批量AI分析任务。

---

## Architecture

### Core Systems

1. **行情数据系统**: `行情源接入` -> `数据清洗` -> `周期聚合` -> `实时推送`
2. **缠论分析系统**: `K线数据` -> `笔段识别` -> `中枢构建` -> `信号生成` -> `AI报告`
3. **策略回测系统**: `策略定义` -> `历史回测` -> `绩效分析` -> `参数优化`
4. **用户管理系统**: `权限控制` -> `自选股管理` -> `分析报告存储` -> `个性化配置`
5. **TradingView对接系统**: `数据请求处理` -> `周期转换` -> `指标计算` -> `图表数据返回`
