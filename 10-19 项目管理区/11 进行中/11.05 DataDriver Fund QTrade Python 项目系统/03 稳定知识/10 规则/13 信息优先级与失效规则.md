# 信息优先级与失效规则

## 默认优先级
1. [[00 当前状态]]
2. [[00 交付记录总览]] / [[00 问题排查总览]] 中最近仍有效的结论
3. [[00 稳定知识总览]] / [[00 决策记录总览]] / [[00 模块地图总览]] / [[00 API 契约总览]]
4. [[00 会话日志索引]] 和 [[10 运行上下文记录]]
5. 旧式摘要笔记和历史零散记录

## 冲突处理
- 当前状态优先于其他卡片。
- 已交付且验证通过的记录优先于历史问题判断。
- 会话日志只提供时间线，不直接覆盖稳定知识。
- 未验证内容必须标记为待验证或临时结论。
