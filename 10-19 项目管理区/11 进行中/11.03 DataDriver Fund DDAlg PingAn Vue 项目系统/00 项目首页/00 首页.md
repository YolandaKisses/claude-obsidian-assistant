- 最低优先级：[[00 会话日志索引]] 和 [[10 运行上下文记录]]
- 如果不同层内容冲突，优先采用高优先级层；低优先级层保留作历史参考，但不作为当前事实。
- 如果 [[00 当前状态]] 与稳定知识冲突，默认以 [[00 当前状态]] 为准。
- 如果怀疑 [[00 当前状态]] 已经过期，必须用最近交付、最近问题结论或最近会话记录交叉确认。

## 使用规则
- 当前有效事实只进“当前状态”。
- 过程性讨论和阶段结论进“会话日志”。
- 稳定结论进入“稳定知识 / 模块地图 / API 契约 / 问题排查”。
- “为什么这样做”单独放到“决策记录”。
- 一次改动的闭环放到“交付记录”。
- 任何已经能归类到页面、API、问题、决策或交付的稳定内容，都不要只留在会话日志里。
