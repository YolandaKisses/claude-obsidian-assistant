# 11.03 DataDriver Fund DDAlg PingAn Vue 当前状态

## 摘要块
- 这篇讲什么：适合优先给人和 AI 看的项目现状快照。
- 当前结论：项目仍处于历史沉淀深、定位成本高的状态，优先需要稳定的模块/API 入口索引。
- 已过期内容：不再把所有当前事实混在历史日志里。
- 下一步看什么：如果是开发任务，先去 [[00 模块地图总览]] 和 [[00 API 契约总览]]。

## 当前目标
- 让项目上下文具备固定骨架，并逐步承接旧主笔记与旧日志里的稳定结论。
- 让后续每次协作都能快速定位到模块、页面和接口域。
- 把会话里反复出现的稳定结论逐步提升为卡片，而不是继续堆在单一日志里。

## 当前有效结论
- 这个项目最适合“页面目录 + API 域”成对查找。
- `metricCenter`、`reportConfig`、`dataReport`、`newReport`、`xbrl` 是最常用的高价值入口。
- 图形化相关需求需要先从使用页面反查组件，不适合直接从组件库盲搜。
- `ReportRenderer` 的树表识别已经确认存在误判风险，普通列表仅有 `id` 时不应直接按树表处理。
- 组合树管理、`select-table`、资源管理上传白名单是近期已经有结论的高价值改动点。

## 当前风险
- 页面命名和目录命名不完全统一。
- `old`、`copy`、`backup` 等历史文件会干扰入口判断。
- 稳定知识目前仍主要散落在旧主笔记和上下文日志中。
- 旧日志已经开始混合“过程记录”和“可复用知识”，后续若不持续迁移，骨架仍会失效。
