# 11.03 报表与数据上报模块

## 摘要块
- 这篇讲什么：报表配置、数据上报、自定义报表相关页面与接口域。
- 当前结论：这里最重要的是页面目录和 `src/api/reportConfig`、`src/api/dataReport` 的对应关系。
- 已过期内容：不再只把报表域作为一个笼统模块名。
- 下一步看什么：如果涉及渲染链路，再进入新报表模块。
- 适用范围：报表配置、复杂报表配置、数据上报、自定义报表。
- 不适用范围：新报表渲染、XBRL、控制台。

## 入口路径
- 报表配置：`src/views/reportConfig/*` -> `src/api/reportConfig/*`
- 数据上报：`src/views/dataReport/*` -> `src/api/dataReport/*`

## 关联页面
- `src/views/reportConfig/reportConfig.vue`
- `src/views/reportConfig/reportConfigComplex.vue`
- `src/views/reportConfig/checkedConf/index.vue`
- `src/views/reportConfig/reportTool/tool.vue`
- `src/views/dataReport/currencyReport.vue`
- `src/views/dataReport/customReportList.vue`
- `src/views/dataReport/customReportNew.vue`
- `src/views/dataReport/customReportSet.vue`

## 关联 API
- 报表配置：`src/api/reportConfig/*`
- 数据上报：`src/api/dataReport/*`
- 细卡入口：[[21 报表配置页面-API映射卡]]、[[22 数据上报页面-API映射卡]]

## 职责边界
- 负责报表模板配置、复杂报表配置、校验配置、报表工具、主报表上报与自定义报表管理。
- 不负责 `newReport` 渲染链路和 XBRL / 控制台业务域。

## 常见修改点
- 配置字段与回显
- 列表查询条件
- 上报参数结构
- 自定义报表配置与提交

## 典型问题
- `reportConfig.vue` 与 `reportConfigComplex.vue` 职责接近，改动前要先区分真实使用场景。
- 常规上报列表页分散，但通常共用 `@/api/dataReport/index`。
