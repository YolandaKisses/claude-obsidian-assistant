# 25 控制台页面卡

## 摘要块
- 这篇讲什么：控制台与服务治理页面入口、职责与修改提示。
- 当前结论：控制台页面通常使用本地 `api/*.js`，也可能混用公共授权接口。
- 已过期内容：不要把控制台页面统一归为“系统管理”而不落到页面文件。
- 下一步看什么：改接口时继续读 [[25 控制台页面-API映射卡]]。
- 适用范围：任务管理、历史监控、平台参数、数据模型管理、服务授权。
- 不适用范围：XBRL 页面、指标中心页面。

## 页面中文名
- 控制台

## 页面路径
- `src/views/control/todayTaskManage/index.vue`
- `src/views/control/historyTaskMonitoring/index.vue`
- `src/views/control/platformParameterSet/index.vue`
- `src/views/control/dataModelManage/index.vue`
- `src/views/control/serviceAuthorization/*`

## 页面职责
- 承接任务管理、历史监控、平台参数设置、数据模型管理和服务授权能力。

## 关联 API
- `src/views/control/*/api/*.js`
- `src/api/serviceAuthorization.js`
- 详见 [[25 控制台页面-API映射卡]]

## 常见修改点
- 查询筛选条件
- 列表展示字段
- 参数配置
- 授权状态和操作入口

## 常见坑
- 同一个目录下有本地 API 和全局 API 混用，改动前要先辨认调用链。
