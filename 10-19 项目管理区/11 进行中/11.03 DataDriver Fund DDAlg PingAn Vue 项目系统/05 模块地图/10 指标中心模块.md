# 11.03 指标中心模块

## 摘要块
- 这篇讲什么：指标中心相关页面、接口和定位策略。
- 当前结论：这是最值得优先结构化的高频模块，目录深、历史版本多、定位成本高。
- 已过期内容：不要只在旧主笔记里保留一个总称。
- 下一步看什么：如果是具体功能排查，再继续拆页面卡。
- 适用范围：指标树、指标配置、指标地图、算法管理、数据模型、指标看板等指标中心能力。
- 不适用范围：报表配置、数据上报、新报表、XBRL、控制台。

## 入口路径
- 页面目录：`src/views/metricCenter`
- API 组织：多数子模块使用各自本地 `api/index.js`

## 关联页面
- 指标树管理：`src/views/metricCenter/metricTreeManagement/index.vue`
- 指标树管理（新）：`src/views/metricCenter/metricTreeManagementNew/index.vue`
- 指标配置：`src/views/metricCenter/metricConfiguration/index.vue`
- 指标地图：`src/views/metricCenter/indexMap/index.vue`
- API 管理：`src/views/metricCenter/apiManager/index.vue`
- 算法管理：`src/views/metricCenter/algManager/index.vue`
- 算法缓存：`src/views/metricCenter/algCache/index.vue`
- 算法查询：`src/views/metricCenter/algQuery/index.vue`
- 数据模型：`src/views/metricCenter/dataModel/index.vue`
- 指标看板：`src/views/metricCenter/idxDashboard/index.vue`

## 关联 API
- 多数子模块使用各自本地 `api/index.js`
- 总览入口：[[00 API 契约总览]]
- 细卡入口：[[20 指标中心页面-API映射卡]]

## 职责边界
- 负责指标树、指标配置、算法、数据模型与指标看板等核心指标平台能力。
- 不负责报表配置、数据上报、新报表渲染、XBRL 与控制台域逻辑。

## 常见修改点
- 表格交互与拖拽
- 树结构展示与节点行为
- 指标配置项字段与回显
- 算法、模型、看板列表逻辑

## 典型问题
- `New`、`copy`、历史版本并存，容易改错真实入口。
- 组合树管理表头拖拽、`select-table` 样式与 `isMultiple` 配置是近期高价值改动点。
