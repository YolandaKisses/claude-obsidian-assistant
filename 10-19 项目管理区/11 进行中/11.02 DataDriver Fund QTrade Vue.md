# DataDriver Fund QTrade Vue

## 相关记录

- 上下文记录：[[11.02 DataDriver Fund QTrade Vue 上下文记录]]

面向基金交易团队的量化交易前端平台，承接行情浏览、TradingView 图表分析、缠论/AI 分析结果展示、自选股与股票池管理等前台交互能力。当前仓库是独立前端工程，主要通过 REST API 与 `DataDriver Fund QTrade Python` 后端联调。

## Architecture

**架构图：[[DataDriver Fund QTrade Vue Architecture.canvas]]**

### 交互主链路

`用户操作` -> `Vue Router` -> `Pinia Store / 本地状态` -> `src/api/* 或页面内请求封装` -> `QTrade Python 后端接口` -> `组件渲染` -> `TradingView / ECharts / AI 报告展示`

### 核心模块

- **行情与股票列表**：股票浏览、筛选、自选管理、刷新策略
- **缠论分析**：缠论页面、缠论扩展页面与图表联动
- **AI 分析**：AI 报告展示、Markdown 内容承载、分析结果交互
- **回测与模拟基金**：策略回测、模拟盘相关页面
- **固收交易**：`firmBargain` 相关页面与组件
- **系统与权限**：基于 dvadmin 体系的系统管理、菜单、权限能力

### 数据流关注点

1. 页面从 `src/views/quantitative/*` 或 `src/views/AI/*` 发起用户动作
2. 通过 `src/api/*`、通用请求封装或页面内业务请求代码调用后端接口
3. 结果进入 `Pinia` 或组件局部状态
4. 图表组件、列表组件和 AI 结果组件消费数据并渲染
5. 在高频刷新场景下，需要重点关注轮询、请求并发和卸载清理

## 开发时的定位建议

- 改页面展示：先找 `src/views/*` 对应页面，再回溯其子组件
- 改图表行为：优先看 `TVChartContainer`、ECharts 组件和页面内图表配置
- 改 AI 结果展示：优先看 `src/views/AI/*` 和 `src/components/stockAnalysisAI`
- 改登录/权限/菜单：优先看 `src/api/login`、`src/api/menu`、路由和 store
- 查不到接口文件时，继续在页面目录内搜 `axios`、`request`、`get`、`post`
